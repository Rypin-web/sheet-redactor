# Sheet Redactor

Утилита для объединения XLSX таблиц, извлечения данных и составления сводных отчётов.

**Версия:** 1.2.3

## Возможности

- 4 сценария обработки данных
- Поддержка старых и новых данных
- Автоматическое открытие результата в Excel
- Выравнивание столбцов по ширине
- Обработка alarm-отчётов (для сценариев с последними сутками)

## Сценарии работы

### 1. Прошлые даты: Обе сравниваемые точки – в одной таблице

Обе точки (А и Б) с данными CCSR и Rate находятся в одной таблице. Данные для Rate берутся из второй таблицы.

**Поток:**
```
Таблица 1 → НАЗВАНИЯ → CCSR → Rate → Точка А → Точка Б →
Таблица 2 → НАЗВАНИЯ → Rate
```

### 2. Прошлые даты: Сравниваемые точки – в разных таблицах

Точка А находится в первой таблице, точка Б — во второй. Обе таблицы содержат CCSR и Rate.

**Поток:**
```
Таблица 1 (Точка А) → НАЗВАНИЯ → CCSR → Rate → Точка А →
Таблица 2 (Точка Б) → НАЗВАНИЯ → CCSR → Rate → Точка Б
```

### 3. Последние сутки: Обе сравниваемые точки – в одной таблице, Rate – в отдельной

Обе точки (А и Б) с данными CCSR находятся в одной таблице. Rate берётся из второй таблицы (самые свежие данные по названию соты).

**Поток:**
```
Таблица 1 → НАЗВАНИЯ → CCSR → Точка А → Точка Б →
Таблица 2 → НАЗВАНИЯ → Rate →
Alarm-table (опционально)
```

### 4. Последние сутки: Сравниваемые точки — в двух разных таблицах, Rate — в третьей

Точка А находится в первой таблице, точка Б — во второй. Rate берётся из третьей таблицы (самые свежие данные по названию соты).

**Поток:**
```
Таблица 1 (Точка А) → НАЗВАНИЯ → CCSR → Точка А →
Таблица 2 (Точка Б) → НАЗВАНИЯ → CCSR → Точка Б →
Таблица 3 → НАЗВАНИЯ → Rate →
Alarm-table (опционально)
```

## Требования

- Windows 10/11
- Node.js 18+ (для запуска из исходников)

## Установка

### Из исходников

```bash
# Клонировать репозиторий
git clone <repository-url>
cd sheet-redactor

# Установить зависимости
pnpm install

# Запустить
pnpm start
```

### Готовый EXE

1. Скачать `sheet-redactor.exe` из релизов
2. Положить в папку с программой файлы `.xlsx` в папку `input/`
3. Запустить `sheet-redactor.exe`

## Использование

1. Положите файлы `.xlsx` в папку `input/`
2. Запустите программу
3. Выберите сценарий из меню
4. Следуйте инструкциям:
   - Выберите таблицы
   - Выберите заголовки столбцов (НАЗВАНИЯ, CCSR/KPI, Rate/ВЕС)
   - Выберите даты точек А и Б
   - Для сценариев 3 и 4: выберите alarm-table или пропустите (0)
5. Результат автоматически откроется в Excel

## Результат работы

Программа создаёт файл `TOP_BAD_CELLS_[timestamp].xlsx` с тремя листами:

1. **Исходные данные** — все записи с вычисленными значениями
2. **Ухудшились** — записи с отрицательным изменением CCSR
3. **ТОП-10** — топ-10 записей по изменению Rate (или по Rate для сценариев 3-4)

### Столбцы результата

| Столбец | Описание |
|---------|----------|
| БС | Имя базовой станции (для сценариев 3-4 с alarm) |
| Название | Имя соты |
| [Дата А] (KPI) | Значение KPI для точки А |
| [Дата Б] (KPI) | Значение KPI для точки Б |
| [Rate] | Вес соты |
| Изменение (KPI) | Разница: Б - А |
| Аварии | Список аварий из alarm-отчёта (для сценариев 3-4) |

## Навигация

- Ввод `r` или `к` на любом шаге возвращает на шаг назад
- Ввод `0` при выборе alarm-table пропускает обработку аварий
- Ввод `r` в главном меню завершает программу
